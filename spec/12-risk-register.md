# 12 — Risk Register

The ranked list of things that can sink mayavius, with the mitigation already
designed into the spec and the **kill-switch** (the cheap fallback we pull when
the mitigation isn't enough). Risks are facts-driven: every external claim traces
to [decisions/decision-log.md](decisions/decision-log.md). This register does not
relitigate locked decisions ([03-decisions-locked.md](03-decisions-locked.md)) —
it protects them.

Scope discipline: the #1 risk is **scope creep into long video** (R1); the #1
quality risk is **looking broken** (R2/R3); the #1 ship-blocker is **licensing**
(R5). Everything below is testable on **the 36 GB Apple-Silicon Mac**; cloud GPU
is an **optional** fallback only, never a prerequisite (handover §3).

---

## 1. Register

Likelihood / Impact scale: **L** low · **M** med · **H** high.

| # | Risk | Likelihood | Impact | Mitigation (designed in) | Kill-switch / Fallback |
|---|------|:---:|:---:|--------------------------|------------------------|
| **R1** | **Long videos** — compute + unified-memory blow-up on MPS and MV4D payload bloat past the shareable ceiling. **The #1 scope risk.** | H | H | Cap clip length at the API boundary: `MAYAVIUS_MAX_CLIP_FRAMES` (default **24**, hard ceiling **≤ 64** = MV4D `T` cap). Decode → **temporally subsample** to the cap (target 32–48) before any model runs ([08](08-dependencies-and-env.md) §6; [05](05-data-contract.md) §4). `MAYAVIUS_MAX_UPLOAD_MB` (64) rejects huge files pre-decode. Asymmetric-compute boundary keeps the cost behind the async job, never on the viewer. | `POST /jobs` **clamps** (subsample to cap) rather than failing when frame count > cap; if upload > `MAX_UPLOAD_MB` or duration is absurd, reject with a clear `4xx` pointing at the cap (no silent no-op — [03](03-decisions-locked.md) conventions). MVP scope is **short clips only** (D-handover §4.6); arbitrary length is explicitly out. |
| **R2** | **Casual-video robustness** — phone footage (motion blur, rolling shutter, hand shake, low light) yields noisy / flickering point clouds. | H | H | Per-point **confidence culling** from VGGT's conf head before encoding (`HAS_STATIC_CONF`, `static_conf` u8 in [05](05-data-contract.md) §3.4); **CoTracker3 `pred_visibility`** drives the TRACKS packed visibility bitmask so low-confidence track samples become ribbon gaps, not jitter. **Temporal smoothing** of track positions (centered moving-average, window=3) in the **core** `ReconstructionService` (`smooth_and_cull`) before encode ([06 §6](06-backend-spec.md)). **Honest UI expectation-setting** — the viewer markets "immediacy + interactivity, not photorealism" (handover §2); a quality/confidence hint surfaces in the result UI. | Graceful degradation: when conf/visibility is uniformly poor, **drop dynamic + tracks and serve a static-only cloud** (still a valid MV4D blob: clear `HAS_DYNAMIC`/`HAS_TRACKS` flags). Curated bundled corpus (D10: 3–4 short CC clips) guarantees a clean first-run demo regardless of user input. |
| **R3** | **Static/dynamic separation fails** — if the whole scene jitters frame-to-frame it reads as "broken," killing the wow. | M | H | The format is **built around this** ([05](05-data-contract.md) §1): STATIC_POINTS is sent **once and rendered every frame** (cannot jitter by construction); only DYNAMIC_FRAMES + TRACKS animate. Separation uses the model's **motion/visibility outputs** — VGGT depth/conf for the stable background, CoTracker3 tracks to identify moving foreground; threshold motion magnitude to assign points static vs dynamic. | If separation is unreliable, **bias toward static**: classify ambiguous points as static (stable, never flickers) and animate only high-confidence movers — a too-still scene beats a jittery one. Optional cleanup: a lightweight segmentation pass (post-MVP adapter step), not an MVP dependency. |
| **R4** | **Point-cloud aesthetics** — raw `THREE.Points` look dotty / sparse / flat; weak screenshots = weak stars. | M | H | Path-1 craft is a first-class deliverable (D-render): custom **vertex/fragment shader** with **depth-based / perspective point sizing** (`gl_PointSize` attenuated by distance), round-sprite alpha, sRGB color handling from the u8 RGB channels, and `Line2`/`LineSegments2` ribbons (anti-aliased screen-space width) for motion. Positions dequantized **in-shader** (no CPU cost) so size/color tuning is free. | Ship sensible **fixed defaults** that look good on the bundled corpus; expose point-size / ribbon-width as tunable uniforms via the viewer store so a bad clip can be salvaged without a rebuild. Path-2 4DGS splats (`Scene.tsx` seam) is the *long-term* aesthetic answer but is **OUT of MVP** — do not pull it as a fallback. |
| **R5** | **Licensing** — all default weights are **non-commercial** (VGGT-1B + cotracker3 = cc-by-nc-4.0) and **no commercial-friendly tracker exists** in the candidate set. | H (certain) | M | **D2**: mayavius **code = MIT**; ship NC weights by default and **label them clearly** — README + `/jobs` metadata surface `AdapterInfo.weights_license` for the active adapter ([06](06-backend-spec.md), [08](08-dependencies-and-env.md) §7). The adapter layer **license-tags every model** so a deployer can gate on it. Honest framing as an open *research* project maximizes stars without misrepresenting redistribution rights. | Commercial host's kill-switch = **`VGGT-1B-Commercial`** (custom `vggt-aup-license`, commercial-OK-except-military, **gated**) for a **static-only** cloud — the **track-ribbon feature stays commercially blocked** until a permissive tracker is sourced. This is a documented limitation, not a bug. |
| **R6** | **MPS op-coverage gap** — a model op is unimplemented on the MPS backend and aborts inference on the Mac. | M | M | Adapters set **`PYTORCH_ENABLE_MPS_FALLBACK=1` before importing torch** ([08](08-dependencies-and-env.md) §5) → unimplemented ops route to CPU (slower, not fatal). VGGT runs via the **community-port pattern** (device `mps`, forced **fp32**, no fp16 autocast); CoTracker3 has **first-class in-tree MPS** (auto-select `cuda > mps > cpu`). The **MPS smoke-test gate** (spec/10) catches a broken op set at build time. | Per-adapter kill-switch: run **that adapter on the optional cloud GPU** ([11](11-deployment-and-launch.md)) via `MAYAVIUS_DEVICE=cuda` — the local path still works for the default VGGT+CoTracker3 combo; only the failing optional adapter moves off-Mac. Never block the MVP on a single op gap. |
| **R7** | **Model download** size/time — multi-GB checkpoints; the commercial checkpoint is **gated** (manual HF access form). | M | M | Default to **ungated NC weights** (`facebook/VGGT-1B`, `facebook/cotracker3`) so local dev never hits a gate. `huggingface-hub` **caches** weights (one-time download; **never committed** — [08](08-dependencies-and-env.md) §8). Surface **download progress** through the job-progress channel (`GET /jobs/{id}` status, 0..1) and a "fetching model" job phase so a cold start isn't a silent hang. | Pre-warm the model cache on first server boot / in the deploy image. Gated `VGGT-1B-Commercial` is **opt-in only** — the executor completes the AUP form out-of-band ([08](08-dependencies-and-env.md) §4.3); it is not on the default install path, so gating never blocks first run. |
| **R8** | **A competitor appears** — fast-moving feedforward-4D space (D4RT, VGGT-Omega, Vista4D, MoRe, TracksTo4D all live or imminent). | M | M | Defensible wedge is the **combination**, not any single component: open + feedforward + point-cloud + **track-ribbons** + **client-only shareable viewer** (no GPU to view) + rich share cards (decision-log §F). Novelty is stated **defensively** ("none surfaced," not "none exists" — absence is unfalsifiable). The swappable `ReconstructionPort` lets us adopt a stronger model (e.g. VGGT-Omega, OpenD4RT) **without rearchitecting** — competitors become adapters. | **Re-run the novelty sweep at launch** (decision-log §F mandate). If a near-identical project ships first, lean into the differentiator that's hardest to copy: the **client-only viewer + shareable permalink** combo (most analogues are transient server-GPU Gradio Spaces with no permalink). |
| **R9** | **torch / MPS version drift** — `torch`/`torchvision` resolve to a build that regresses or breaks MPS for these models. | M | M | **Install-then-freeze** ([08](08-dependencies-and-env.md) §4.1): resolve `torch`/`torchvision` on the target Mac, verify `torch.backends.mps.is_available()`, then **pin the resolved versions into `requirements-ml.txt`**. Validated floor `torch >= 2.5.0` (stable MPS autocast), expected `2.12.0`. The **MPS smoke-test gate** (spec/10) is the regression tripwire on every env build. ML deps are **deferred** and isolated from the lightweight runtime ([08](08-dependencies-and-env.md) §3). | Pin **back** to the last-known-good frozen `requirements-ml.txt` (committed, weights-free). If a new torch is mandatory and regresses MPS, run inference on the cloud GPU ([11](11-deployment-and-launch.md)) until the upstream fix lands. The lightweight backend (FastAPI, no ML) is unaffected by torch drift. |
| **R10** | **Payload size > shareable** — the MV4D blob exceeds a fast-load / linkable size, breaking the share-card virality surface. | M | M | **MV4D caps are part of the contract** ([05](05-data-contract.md) §4): `T ≤ 64`, `N_static ≤ 150k`, dynamic `≤ 20k`/frame, `M ≤ 4096`, **uncompressed target ≤ 12 MB** (hard ceiling 24 MB). Compactness is structural: **16-bit quantized positions** (dequant in shader) + **u8 RGB** + **packed visibility bitmask** + zero-copy SoA. The encoder **enforces caps** (confidence-cull / subsample) and **logs actual size**. Result served with **HTTP compression** (brotli/gzip) + long-lived immutable cache header → shareable permalink. | Encoder over-ceiling kill-switch: **escalate culling** — tighten confidence threshold, drop dynamic points first, then reduce `M`, then subsample `T` further — until under the ceiling, logging what was dropped. (SSE progress stream is **not** GZip-middleware compressed — [08](08-dependencies-and-env.md) §3; compress the result blob at the route level only.) |
